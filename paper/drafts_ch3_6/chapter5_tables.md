## Table 5.4 Limitations and Implications

| Limitation | Affected component | Implication | Possible mitigation | Evidence or citation status |
| :--- | :--- | :--- | :--- | :--- |
| Complete six-model coverage is limited to LastFM and ML-1M. | Comparative scope | General claims must be anchored in the two complete datasets. | Complete additional model-dataset ports under the same validation contract. | Supported by `thesis_analysis_pack/validation_status_table.md`. |
| Amazon-Book KGAT is partial and has no approved explanation alpha sweeps. | Boundary-case analysis | Amazon cannot support a complete six-model trade-off conclusion. | Resolve blocked ports and define approved timestamp, SEP, and ETD denominator protocols. | Supported by `reports/tables/amazon_classic_port_readiness.json`; completion evidence. |
| LIR depends on valid interaction timestamps and recoverable path anchors. | Explanation metric | Missing or sentinel timestamps prevent a valid temporal interpretation. | Require timestamp-semantic validation before reporting LIR. | Internal implementation evidence; primary external citation requires manual check. |
| SEP depends on KG degree as a popularity or serendipity proxy. | Explanation metric | Degree-derived movement does not capture every meaning of novelty or unexpectedness. | Report the operational definition and test alternative proxies in future work. | Internal implementation evidence; primary external citation requires manual check. |
| ETD depends on a declared path-type taxonomy and denominator. | Explanation metric | Scores are dataset- and taxonomy-dependent. | Register and validate the legal relation set for each dataset. | Internal implementation evidence; primary external citation requires manual check. |
| Native-path candidate exhaustion can produce short or empty top-k lists. | Strict accuracy and path fidelity | Missing slots count as non-hits, so accuracy can reflect path availability as well as ranking quality. | Report slot coverage and short-user counts; do not pad with post-hoc non-path items. | Supported by the canonical dataset standard and evaluator. |
| No user study is included. | Human-centred explanation quality | Computational path metrics do not establish perceived usefulness, clarity, or persuasiveness. | Add a separately designed user evaluation. | No user-study artifact identified in the evidence pack. |
| Statistical significance evidence is absent from the registered pack. | Result interpretation | Reported differences are descriptive rather than inferential. | Register repeated-run or significance-test evidence before making inferential claims. | Requires manual check. |
| Results depend on model-port, canonical-ID, export, and validation quality. | Framework execution | BLOCKED can indicate contract incompleteness rather than model failure. | Maintain validation-first reporting and provenance manifests. | Supported by validation and Amazon readiness records. |
| Native and post-hoc explanation scopes are not interchangeable. | Model scope | Non-path recommenders cannot receive faithful LIR, SEP, or ETD scores through post-hoc path recovery. | Restrict main explanation scoring to auditable native-path outputs. | Supported by native-path architecture and candidate audit. |
| Some model and metric citations remain provisional. | Literature support | Mechanism origins cannot be treated as fully verified bibliography. | Verify primary sources, venues, and DOI metadata before submission. | UCPR, KGGLM, and metric sources unverified; other seeds are Medium confidence. |
