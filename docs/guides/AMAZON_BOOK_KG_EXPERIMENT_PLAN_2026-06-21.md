# Amazon-Book Knowledge-Graph Experiment Plan

Date: 2026-06-21

## Decision

Only one larger Amazon dataset is required after the Beauty correctness proof.
The selected dataset is the processed **Amazon-book** benchmark released with
KGAT and reused by later KG recommendation work such as KGIN.

This replaces the earlier proposal to run both Cellphones and Electronics.
Those Amazon review categories contain rich item metadata, but they do not
ship as the same established recommendation benchmark with an independent KG.
Constructing brand/category/co-view edges ourselves would be a useful derived
graph experiment, but it must not be presented as evaluation on a dataset with
an existing native benchmark KG.

## Meaning of “Native KG”

Amazon itself does not publish the review corpus as a knowledge graph.
For this project, “native KG dataset” means:

1. the public recommendation benchmark already distributes user-item
   interactions, item-entity mappings, relation mappings, and KG triples;
2. KG recommenders consume those triples directly in their published
   experimental protocol;
3. this project does not manufacture a graph from review metadata merely to
   make a path model runnable.

The Amazon-book KG was produced by the KGAT data pipeline following KB4Rec:
items were mapped to Freebase entities through title matching where a mapping
was available. It is therefore a benchmark-provided Freebase KG, not an
Amazon-authored KG and not a graph generated by this repository.

Official implementation:

```text
https://github.com/xiangwang1223/knowledge_graph_attention_network
```

## Published Dataset Scale

The KGAT release reports:

- users: 70,679;
- items: 24,915;
- interactions: 847,733;
- KG entities: 88,572;
- relation types: 39;
- KG triples: 2,557,746.

Compared with `beauty_legacy_v1`:

- Beauty interactions: 198,502;
- Amazon-book interactions: 847,733;
- Amazon-book has about 4.3 times as many interactions;
- its independent KG has more than 2.5 million triples.

Beauty remains the cross-model correctness reference. Amazon-book is the
single larger Amazon/KG experiment used to demonstrate scale.

## Canonical Source Contract

The imported source bundle must include and preserve provenance for:

```text
train.txt
test.txt
user_list.txt
item_list.txt
entity_list.txt
relation_list.txt
kg_final.txt
```

The canonical builder must verify:

1. remapped user and item ids are unique and in range;
2. every recommendation item has the documented entity mapping;
3. every KG triple references valid entities and relations;
4. train and test pairs are disjoint;
5. source row counts match the imported manifest;
6. no interaction-derived edges are added to the source KG.

The published source has train/test but no validation split. The canonical
version will deterministically hold out one interaction per eligible user
from source train for validation. This requires retraining every model; source
checkpoints and reported metrics are provenance references only.

## Model Scope

Primary native-path models:

- KGGLM;
- PEARLM;
- PGPR;
- UCPR;
- CAFE.

Accuracy-only appendix references:

- KGIN;
- LightGCN on the same canonical interactions;
- KGAT after the DGL dependency blocker is resolved.

The path models require a reproducible projection of the 39-relation Freebase
KG into their typed action spaces. The projection must retain original
relation ids/names and entity ids in remap tables. It may prune relations for
bounded path search, but it may not replace the benchmark KG with metadata
edges.

## Required Gates

Before full model runs:

1. source download and checksums pass;
2. canonical train/validation/test labels round-trip exactly;
3. KG entity/relation/triple counts pass;
4. model-specific KG projections publish retained/dropped relation counts;
5. no validation/test interaction is introduced into the KG;
6. preprocess and model initialization smokes pass;
7. one-user and 100-user path inference/export smokes pass;
8. full inference uses resumable user shards and deterministic merging;
9. all exported uid/pid values map back to canonical ids.
